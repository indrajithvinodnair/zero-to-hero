One way data binding

This essentially means that the data flow takes place only in one direction
That is either from View to Model or from Model to View and never simultaniously

Interpolation

The value of the text variable diplayed in this paragraph is populated using string Interpolation
of the text variable stored in component

The one important thing about string interpolation is that eveything within the curly braces are
considered as string. The type of the variable defined in the component doesnt matter
String interpolation always considers the content within the curly braces as string elements

Property Binding

Property binding is used to set values for the attributes of target elements

The syntax is [value]="variable"
The value can by any valid property of an element

Here is the username field have the deault value of user, it is populated using the username variable in the component


The advantage of using property binding is that the values
are considered to be of their original data type.br
That is, if there is a boolean variable in the component, it can be used to set value for
a components propertywhich accepts boolean values .
Where as if we were to use string interpolation, then the values would sure be considered as string only.

The text box shown below is set as disabled by setting the disabled property as true using a boolean variable

Event Binding

The button calls an displayMsg() when it is clicked

Syntax is (click)="function()"
